IPX/SPX

  • IPX = Internet Packet Exchange - zajišťuje přenos paketů bez potvrzování mezi aplikacemi připojenými na zvolené připojovací místa (Socket)
  • SPX = Sequential Packet Exchange - nadstavba IPX zajišťující potvrzování přenesených paketů
  • Vycházejí ze systému XNS (Xerox Network System), který byl alternativou firmy Xerox k protokolům TCP/IP.
  • Adresy v IPX jsou definovány jako dvojice 32-bitová adresa sítě a 48bitová adresa stanice.
    • Podstatnou nevýhodou IPX/SPX je skutečnost, že adresu sítě definuje správce konkrétní sítě. Chybějící kooperace v přidělování adres v principu znemožňuje vzájemné propojení sítí pod protokoly IPX/SPX mezi sebou.
  • Implementuje 4bitový HOP count - překročení limitu šestnácti směrovačů je důvodem k likvidaci paketu
[editovat]

NetBIOS

  • Nejstarší síťový protokol určený pro prostředí lokální sítě.
  • Navržený firmou IBM.
  • Základní komunikační protokol do struktury sítě MS-Net.
  • Rozhraní NetBIOSu nabízí čtyři skupiny funkcí
    • správa tabulek jmen
    • datagramové služba - dovoluje předání zprávy o délce do 512 B jednomu adresátu nebo její broadcast
    • služba virtuálních kanálů - neboli taktéž relace - dovoluje přenášet zprávy o délce 131071 znaků, které jsou při přenosu děleny do paketů
    • pomocné funkce
  • Komunikující aplikace jsou identifikovány jmény o šestnácti znacích

Správa Lokálních Sítí

Je potřeba zajišťovat stavové informace týkající se aktivních prvků, oznamovat jejich výpadky a chyby na médiu, měřit zatížení sítí a segmentů a následně soustředit tyto údaje pro potřebu správce sítě v jediném místě.

[editovat]

Analyzátor Provozu

  • Dnes většinou osobní počítač
  • Obvykle vybaven vestavěným sériovým rozhraním schopným analyzovat dvoubodové spoje rozsáhlých sítí např. Asynchronní + synchronní kanály, rozhraní X.25, ISDN, ...
  • Síťové rozhraní - přijímá rámce i pro ostatní počítače(promiskuitní režim), ale i rámce neúplné, rámce poškozené kolizí a rámce ve kterých byla indikována chyba.
  • Specializovaný SW - analyzuje údaje o poškozených rámcích, určování zdrojů problémů, třídění podle protokolů, komunikujících účastníků, ...
[editovat]

SNMP

  • SNMP = Simple Network Management Protocol
  • SNMP agent = modul správy na ovládaném zařízení
  • SNMP manager / správce SNMP = program pro správu sítě
  • Základem komunikace je databáze MIB
  • Strukturou se blíží ISO CMIS/CMIP.
  • Pracoviště zprávy získává informace od ovládaných zařízení tak, že jim zasílá požadavky Get Request nebo Get Next Request s identifikací MIB prvku a dostává odpovědi Get Response obsahující příslušnou hodnotu. Pro změny hodnoty je tu Set Request a krome toho může ovládáné zařízení hlásit asynchronně výjimečné situace zprávou Trap.
[editovat]

RMON

  • Remote Monitor
  • Sledování provozu v jednotlivých kolizních doménách
  • Bývá součástí aktivních prvků
  • O objekty RMON byla rozšířena databáze MIB
