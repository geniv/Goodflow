Databázová mateřština SQL

[Zpět na kategorii]

Datum: 25. 2. 2008 0:50       Autor: Tomáš Jecha       Zobrazeno: 2130x

Kategorie: Databáze

Témata: SQL

Seriál: Stopařův průvodce po databázích - Díl 2.

Po delší odmlce přináším druhý díl seriálu o databázích řešených Microsoft SQL Serverem. Ukážeme si základy SQL dotazovacího jazyka, jeho zpracování v SQL Server Management Studiu a zároveň rozšíříme tabulky našeho ukázkového internetového obchodu z prvního dílu.

Co je to SQL?

SQL je akronym anglického výrazu Structured Query Language - strukturovaný dotazovací jazyk. Jedná se o textový jazyk navržený ke správě databázových systémů. Rozebírat tu jeho historii a vývoj by bylo nošením dříví do lesa, o tom si můžete přečíst třeba na Wikipedii.

SQL vs T-SQL

Už v minulém díle jsem psal, že T-SQL (Transact SQL) je rozšíření jazyka SQL, kterou využil Microsoft v jeho SQL Serveru, jímž se tento seriál zaobírá. I když bude velká část funkcí fungovat (nebo se alespoň jejich zápis v různých odrůdách SQL jazyku sobě podobat), tak 100% kompatibilní rozhodně nejsou. Ani by to nebylo možné, každý databázový systém totiž řeší různé problémy jinak a i jejich vývoj se ubírá jiným směrem. Přinejmenším nám to ale usnaďnuje přestup z jednoho systému na druhý a databázový vývoj to značně zjednodušuje. V mém seriálu budu ale využívat jen T-SQL a proto příklady třeba v MySQL (či jiných systémech) fungovat pravděpodobně nebudou.

Základy SQL jazyku a orientace v něm

Každý příkaz je složený z klíčových slov, jmen proměnných, statických hodnot atp. Ostatně jako ve většině normálních počítačových jazyků. Když nebudeme zatím brát v potaz příkazy, které přímo neovliňují data (například na větvení programu atp.), můžeme si rozdělit dotazy na tři základní skupiny:

  • Příkazy pro práci s databázovými objekty
  • Příkazy pro práci s daty
  • Příkazy pro dodatečné nastavování

Příkazy pro práci s databázovými objekty

Databázovým objektem rozumíme prakticky vše co definuje databázi a její chování. Od datových tabulek, přes uživatelská oprávnění až po samotné databáze v databázovém serveru. Dotazy, které s nimi pracují začínají jedním z těchto klíčových slov:

  • CREATE - vytvoření objektu
  • ALTER - změna objektu
  • DROP - smazání objektu

Následuje pak klíčové slovo označující typ objektu a ve většině případech i jeho jméno (takže například vytvoření tabulky začíná "CREATE TABLE JmenoTabulky", smazání uživatele "DROP USER JménoUživatele" atp.).

Bližší syntaxe jednotlivých formulací si vysvětlíme později, důležité je, že se jedná o příkazy, které formulují chování databáze a proto je budeme využívat v převážné většině případů jen při jejím návrhu. Je to z toho důvodu, že tabulky jsou optimalizované pro co nejrychlejší čtení, vyhledávání atp., proto mějte na paměti, že dynamické vytváření a mazání třeba tabulek nemusí být nejrychlejší a pro server v běžném produkčním prostředí to může být dost silná káva a můžete tím dosáhnou značného zpomalení.

Příkazy pro práci s daty

Pro koncovou aplikaci je však mnohem zajímavější druhá skupina dotazů, která umožňují číst (příkaz SELECT), měnit (UPDATE), mazat (DELETE) a vkládat (INSERT) datové řádky do tabulek. Obecně se používají k práci s hotovou databází a budeme je intenzivně zkoumat již v tomto díle.

Příkazy pro dodatečné nastavování

Nejprve jsem zvažoval, zda mám tuto skupinu v tomto základním výčtu vůbec uvádět. Nakonec jsem se rozhodl pro především kvůli četnosti jeho použití a navíc se často stává řešitelem problémů z velmi nepříjemného soudku označeného jako "proč to tam nefunguje, když tady ano".

Všechny tyto příkazy začínají slovem SET a mění chování určitých příkazů proti jejich implicitnímu nastavení. Kámen úrazu je v tom, že právě ono implicitní nastavení může být v určitých případech ovliněno řadou věcí, které nelze obecně předvídat. Jedná se například o formát datumu, číslo prvního dne v týdnu, zápis desetinné čárky, jazyk, izolační level transakcí atp. Opět bych zatím vše nerozebíral do hloubky (pro nedočkavé je tu odkaz na oficiální popis příkazu SET v angličtině), zatím nám bude totiž stačit znát základní význam tohoto klíčového slova - modifikace nastavení určitého chování serveru v právě otevřené relaci.

SQL jazyk a Microsoft SQL Server Management Studio

Nejsem zastánce dlouhé teoretické průpravy. I proto si teď ukážeme jakým způsobem budeme příkazy na databázi testovat. Už v minulém díle jsme k pohodlné komunikaci používali SQL Server Management Studio (dále jen SSMS) a není důvod proč tohoto výborného nástroje nevyužít i k testování obyčejných příkazů.

Spustíme SSMS, přihlásíme se a začneme klepnutím na tlačítko . Zobrazí se nám nová lišta (tu si v dalším odstavci popíšeme) a okno pro zápis SQL dotazu a zobrazení následného výsledku:

Lišta SQL dotazu

První věc kterou si musíme uvědomit je, že dotaz voláme vždy proti jedné určité databázi na databázovém serveru. To, kterou lze snadno vyčíst a změnit pomocí rozbalovacího seznamu v liště (na obrázku je to nyní eshop). Na levo od ní nalezneme navíc tři ikonky () sloužící k připojení, odpojení a změně databázového serveru. Napravo pak tyto tlačítka:

  •   - spuštění dotazu proti databázi (také lze spustit pomocí F5).
  •   - verifikace syntaxe - upozorní na některé případné syntaktické chyby v kódu.
  •   - zastavit provádění dotazu (například při nekonečné smyčce).
  •   - spustí dotaz a zobrazí Execution Plan (o něm ale až později - jedná se o vizuální zobrazení provedených operací pro pokročilou optimalizaci dotazů).
  •   - návrhář dotazu - spustí vizuálního návrháře dotazu (funguje jen na základní čtecí příkazy SELECT) a dovolí nám pomocí něj vytvářet základní propojení tabulek, filtrování a řazení (o tom si v tomto článku také ještě povíme).
  •   - editor šablon - celkem nedůležitá funkce pro nahrazování šablonových výrazů - zmíním se o ní až bude nějaká možnost ji využít.
  •   - tyto tři dodatečné možnosti nabízejí: zahrnutí execution plan do výsledku, zahrnutí statistik spojení se serverem a povolení tzv. SQLCMD modu (režim příkazové řádky).
  •   - můžete si vybrat jeden ze 3 druhů výstupů: textový, do tabulky a do souboru. Nejedná se o nic jiného než o způsob jak se zobrazí výsledek. Implicitně je nastavený výstup do tabulky, což je optimální pro naše potřeby.
  • - stejné jako ve Visual Studiu - zakomentování bloku kódu / odkomentování a změna odsazení.

Funkce tlačítek asi nebudeme dále rozebírat a podíváme se na základní příkazy, které se nám budou hodit.

Základy SQL

V prvním díle jsme vytvořili tabulku ukázkové databáze obsahující seznam výrobců. Tu si teď trochu rozšíříme. Jen k tomu nebudeme využívat vizuální návrhář, ale SQL jazyk.

Ať máme jistotu, že všichni mají stejné podmínky, tu původní nejdříve smažeme (tedy pokud jste ji zakládali). Využijeme na to příkaz na zrušení databázového objektu DROP, klíčové slovo označující tabulku TABLE a unikátní jméno tabulky - Companies:

DROP TABLE Companies

Jak tabulku dotazem smazat si můžete prohlédnout na následujícím videu. Nejdříve v něm databázi otevírám, prohlížím přítomnost tabulky Companies a pak vyvolávám dotaz. Když poté dám refresh tabulek, je vidět, že již v databázi není (proto při druhém volání vyskočí chybová hláška, že takový objekt neexistuje, protože je již smazaný):

Ted si vytvoříme tabulku novou, rozsířenou o několik sloupců (City - město, kde firma podniká a Website - url adresa webu firmy). Dotaz na vytvoření je následující, vložte ho tedy do okna dotazu a spusťte (jen si dejte pozor a nazapomeňte spustit dotaz proti naší databázi eshop - tu jsem si vytvořili v prvním díle):

CREATE TABLE dbo.Companies
    (
    CompanyId int NOT NULL IDENTITY (1, 1) PRIMARY KEY,
    CompanyName nvarchar(100) NOT NULL,
    CompanyDescription nvarchar(500) NULL,
    City nvarchar(100) NOT NULL,
    Website varchar(100) NULL
    )

Jestli se vše provedlo správně, objeví se hláška:

Command(s) completed successfully.

Vytváření jednoduchých tabulek pomocí příkazu CREATE TABLE

Nyní si o dotazu povíme něco bližšího. CREATE TABLE znamená, že se vytvoří databázový objekt typu tabulka. Následuje jméno - dbo.Companies. Asy bych měl vysvětlit co znamenají ty 3 písmenka dbo - jedná se o identifikaci databázového schématu. Pokud to nenastavíme jinak, bude se vždy využívat základní, které je právě pojmenované jako dbo (database object - databázový objekt). Toto schéma by se využilo i kdyby jsme ho neuvedli. Schémata obecně slouží k rozdělení databáze do ucelených skupin a to jak funkcionálních, tak bezpečnostních. Nám bude stačit zatím pouze jedno a tak se jimi teď nebudeme zaobírat.

Další parametry příkazu jsou uzavřené v závorkách a definují sloupce tabulky oddělené čárkou. První je unikátní jméno sloupce (unikátní v rámci tabulky), následuje datový typ a přepínač NULL (sloupec může obsahovat prázdnou hodnotu) nebo NOT NULL (sloupec nemůže obsahovat prázdnou hodnotu). Důležité je, že databáze může obsahovat tzv. hodnotu NULL u všech datových typů - to například znamená rozdíl mezi číslem 0 a hodnotou NULL nebo prázdným textem a NULL.

Dále můžeme definovat řadu doplňujících informací oddělených mezerou, uvedu několik z nich:

  • PRIMARY KEY - v tabulce může být jen jednou - označuje klíčový sloupec (viz v příkladu sloupec CompanyId)
  • IDENTITY(1,1) - automatická inkrementace klíče při vkládání nových řádků (opět v příkladu sloupec CompanyId)
  • DEFAULT(hodnota) - máme možnost definovat defaultní hodnotu sloupce, pokud ho neurčíme při vkládání nového řádku

Tímto krátkým popisem se snažím vysvětlit základ vytváření tabulek v T-SQL - bližší informace lze vyčíst na adrese http://msdn2.microsoft.com/en-us/library/ms174979.aspx nebo z dalších pokračování tohoto seriálu.

Vkládání dat do tabulek pomocí příkazu INSERT

Tabulku máme vytvořenou a neměl by nyní být problém do ní pohodlně vložit řádky pomocí nástroje SSMS. To si klidně vyzkoušejte. Uvidíte mimo jiné i nutnost zadat sloupce označené jako NOT NULL, dodržování datových typů a automatické generování klíče do sloupce CompanyId.

Teď je ale čas si ukázat využití příkazu INSERT. Tady je ukázkový dotaz na vložení řádku s fiktivní firmou:

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, City)
    VALUES (N'Dveřní klepátka s.r.o.', N'Zakázková výroba dveřních klepátek a zvonků', 'Brno')

Příkaz začíná INSERT INTO (vlož do). Následuje jméno tabulky a v závorce výčet sloupců, do kterých budeme dosazovat (oddělených čárkou). Za závorkou následuje klíčové slovo VALUES (hodnoty) a za ním v závorce výčet hodnot. Počet hodnot musí být pochopitelně stejný jako počet zadávaných sloupců.

Při vkládání se navíc kontroluje integrita proti pravidlům definovaných při vytváření tabulky. Tedy například pokud nezadáte sloupec City, jenž je označen jako NOT NULL, zobrazí se chybová hláška:

Cannot insert the value NULL into column 'City', table 'eshop.dbo.Companies'; column does not allow nulls. INSERT fails.

V překladu: Nelze vložit NULL hodnotu do sloupce 'City', do tabulky 'eshop.dbo.Companies'; sloupec nedovoluje hodnoty NULL. Vkládání se nezdařilo. Nezadáním hodnoty se totiž automaticky vkládá hodnota NULL.

TIP - Zapisování řetězců: Pro zápis řetězců do SQL kódu se využívají apostrofy - znak ' . Například 'Ahoj'. Když chcete zapsat apostrof do textu, musíte ho zdvojit. Například 'Ahoj ''kdo'' jsi?' znamená Ahoj 'kdo' jsi?. V případě, že napíšete před začátek textu velké písmeno N, bude se text brát jako unicode - tedy vhodné pro všechny datové typy jako je nvarchar, nchar atp. Ovšem není to pravidlo, je to jen doporučení při zápisu speciálních znaků jako písmena s diakritikou, které by se mohly špatně zinterpretovat atp. Tedy například:  N'Příliš žluťoučký kůň úpěl dábelské ódy'.

Aby jsme toho měli v tabulce víc, spusťte také tyto dotazy na vložení dalších řádků:

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, Website, City)
    VALUES (N'Záchodové mísy k.s.', N'Výroba prvotřídních mís', NULL, 'Brno')

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, Website, City)
    VALUES (N'Windows User Group Liberec', N'Pořádání meetingů pro odbornou veřejnost', 'http://www.wug.cz', 'Liberec')

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, Website, City)
    VALUES (N'Eviny pletky', N'Fešné futrálky na nejrůznější věci', 'http://evinypletky.ic.cz/', 'Liberec')

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, Website, City)
    VALUES (N'Severočeská magnetárna', N'Výroba prvotřídních magnetů', NULL, 'Liberec')

INSERT INTO dbo.Companies (CompanyName, CompanyDescription, Website, City)
    VALUES (N'Pražské houslárny', N'Housle pro děti i dorost', NULL, 'Praha')

Příkaz SELECT slouží ke čtení dat. V prní ukázce si necháme vypsat všechny řádky z tabulky Companies:

SELECT * FROM dbo.Companies

Začínáme slovem SELECT, pokračujeme výčtem sloupců oddělené čárkou, které chceme vypsat (hvězdička zastupuje všechny sloupce najednou). Následuje FROM a jméno zdrojové tabulky.

Tak lze zapsat příkaz SELECT ve své základní podobě. Jeho rozšiřování si popíšeme v následujících kapitolách.

Filtrování výsledků pomocí klíčového slova WHERE

Je velmi pravděpodobné, že budeme chtít sady výsledků filtrovat. Používá se k tomu převážně klíčové slovo WHERE. Chceme například zobrazit jen firmy z Liberce:

SELECT * FROM dbo.Companies WHERE City = 'Liberec'

Za WHERE se zapisuje logický výraz určující, zda bude řádek obsažen ve výsledku. Výrazů můžeme použít kolik uznáme za vhodné a na jejich spojování jsou určeny logické operátory. Například výpis firem z Liberce nebo Prahy :

SELECT * FROM dbo.Companies WHERE City = 'Liberec' OR City = 'Praha'

Pro srovnávání jsou aplikovatelné základní operátory:

  • = - rovná se
  • > - větší
  • < - menší
  • >= - větší nebo rovný
  • <= - menší nebo rovný
  • <> - nerovná se

Myslím, že není potřeba je vysvětlovat. Mnohem zajímavější jsou logické operátory. Těch je celá řada:

  • AND - [výraz_1] AND [výraz_2]
    Musí být splněné obě logické části.
  • OR - [výraz_1] OR [výraz_2]
    Stačí aby byl splněný alespoň jeden výraz.
  • NOT - NOT [výraz]
    Neguje výsledek výrazu.
  • BETWEEN - [hodnota_1] BETWEEN [hodnota_2] AND [hodnota_3]
    Platí pouze pokud bude [hodnota_1] v rozsahu [hodnota_2] a [hodnota_3]. Stává se výhodným například při filtrování podle rozsahu datumů nebo ceny. Je to ekvivalent zápisu:
    ([hodnota_1] >= [hodnota_2] AND [hodnota_1] <= [hodnota_3])
  • IN - [hodnota_1] IN ([hodnota_2], [hodnota_3], [hodnota_4]...)
    Platí v případě, že [hodnota_1] odpovídá alespoň jedné z hodnot ve výčtu v závorce. Ekvivalent zápisu:
    ([hodnota_1] = [hodnota_2] OR [hodnota_1] = [hodnota_3] OR [hodnota_1] = [hodnota_4] ...)
  • ALL - [hodnota_1] > ALL ( [dotaz] )
    Dokáže provést hromadné srovnání libovolným srovnávacím operátorem (v ukázce použit jen >) se sadou výsledkou pomocí dalšího dotazu uvedeného v závorce. Je potřeba aby výsledek dotazu v závorce vracel pouze jeden sloupec, se kterým se provede srovnání. Výsledek výrazu bude pozitivní pouze v případě že bude srovnání se všemi řádky z dotazu pozitivní.
  • ANY (popřípadě ekvivalent SOME) - [hodnota_1] > ALL ( SELECT Cena FROM Vyrobky )
    Prakticky to samé co dělá ALL, jen s tím rozdílem, že stačí aby srovnání proběhlo pozitivně pro kterýkoliv z řádků poddotazu ze závorky.
  • EXISTS - EXISTS ( [dotaz] )
    Platí tehdy, když dotaz v závorce vrací alespoň jeden řádek.
  • LIKE - [hodnota_1] LIKE 'wildcards_maska'
    Využijte tzv. wild-cards masku pro srovnání s textovou hodnotou. Wildcard znaky jsou: % (zastupuje libovolný počet libovolných znaků), _ (zastupuje jeden libovolný znak), [abcde] (zastupuje jeden libovolný znak, který je ve výčtu v hranaté závorce) a [^abcde] (zastupuje jeden libovolný znak, který není v hranaté závorce). Lepší bude asi malá ukázka:
    Pro masku 'Ah%' odpovídají všechny řetězce začínající na Ah. Například: 'Ahoj', 'Ah432', 'Aha' atp. Pro '_hoj' zase všechny, které mají 4 písmena a končí hoj. Třeba: 'Ahoj', 'Xhoj', '5hoj' atp.
    Klasické použití je pro vyhledávání. Například všechny firmy, jejichž jména měst začínající na L se získají dotazem: 
    SELECT * FROM Companies WHERE City LIKE 'L%'

Řazení výsledků pomocí klauzule ORDER BY

Celkem jednoduchá funkce je řazení. Dělá se to zapsáním klíčových slov ORDER BY na konec dotazu, které následuje jméno sloupce / sloupců podle nichž se bude řadit. Například:

SELECT * FROM Companies WHERE City = 'Liberec' ORDER BY CompanyName

Výsledek se nám seřadí podle jména firmy.

Navíc je možné přidat další sloupce, které se využijí při shodě hodnot prvního řadícího sloupce. Takové se přidávají za oddělovací čárku.

Další možností je určování směru řazení pro kterýkoliv ze zahrnutých sloupců. Implicitně je to od nejmenšího po největší (ASC), lze však směr obrátit slovem DESC zapsaným za jméno sloupce. Ukázka jest řazení firem podle jména měst a při shodě měst podle jména firmy od největší po nejmenší (větší se bere u řetězců ten s vyšší ordinální hodnotou znaků z leva do prava - řazení podle abecedy):

SELECT * FROM Companies ORDER BY City, CompanyName DESC

Mazání řádku z tabulky pomocí DELETE

Teď už umíme řádky vybírat a filtrovat. Mazání pracuje na podobném principu - smažou se řádky odpovídající kritériím. Používá se příkaz DELETE FROM (smaž z). Ukázka by měla vše vysvětlit - maže všechny z Brna:

DELETE FROM Companies WHERE City = 'Brno'

V případě, že klauzuli WHERE vůbec neuvedete, smažou se všechny řádky - stejně jako u SELECTu, kde se bez WHERE vypíší také všechny.

Modifikace řádku pomocí UPDATE

Poslední příkaz, co si dnes probereme je modifikační příkaz. Začíná UPDATE, pokračuje jménem tabuly a klauzulí SET určující sloupce a jejich nové hodnoty. Na konec se uvádí podmínky změny nám již známým příkazem WHERE. Opět, když ho nezapíšete, zmodifikují se všechny řádky.

Ukázka mluví za vše (nastavíme popis všech firem z Liberce na 'Toto je firma z Liberce!'):

UPDATE dbo.Companies SET CompanyDescription = 'Toto je firma z Liberce!' WHERE City = 'Liberec'

Tabulka, pak obsahuje asi nepřekvapující hodnoty:

TIP - Upravování řádků: Sloupců, které se mají změnit lze určit víc (třeba: UPDATE Companies SET City = 'Moskva', Website = 'http://www.adresa.ru' WHERE ...). Navíc je možné upravovat pomocí hodnoty sloupce, kterou obsahoval dříve - hezká ukázka je například inkrementace čísla - UPDATE Tabulka SET PocetPristupu = PocetPristupu + 1 WHERE ...

Závěrem - WHERE a klíče tabulky

Klasickou ukázkou využití klíče tabulky je upravování nebo mazání právě jednoho určitého řádku, který je jím jednoznačně identifikovatelný. Jako klíč se obecně doporučuje využívat buď datový typ čísla nebo GUID identifikátoru. V obou případech platí, že klíčový sloupec se indexuje a tak je vyhledávání podle něj velmi rychlé a to je další důvod proč ho využívat.
